How the Internet Works: (3.1.19)

Chapter 3– Link Layer

“Lowest layer”  > closest to the physical network media

Transmits data using a wire, a fiber optic cable, or a radio signal

Only transmits part of the way (ie. wifi & cellular phone network)

BROADCAST – Sending a packet in a way that all the stations connected to a local area network will receive the packet

GATEWAY – first router that handles your computer’s packets, connecting it to a wider area network (internet). Computers that want to send data outside the local network must send their packets to the gateway for forwarding.

BASE STATION – another word for the first router that handles your packets as they are forwarded to the internet

MEDIA ACCESS CONTROL (MAC) – an address that is assigned to a piece of network hardware when the device is manufactured

CARRIER SENSE – first listen for a transmission, wait for a transmission to finish (if one occurring), then transmits

  • Carrier Sense Multiple Access with Collision Detection (CSMA/CD) – listen, transmit, listen, and wait and retry if necessary
  • Best suited when medium is inexpensive, shorter distance, and there are a lot of stations sharing the medium – sending data in short bursts

TOKEN style network – a technique to allow many computers to share the same physical media without collisions. Each computer must wait until it has received a token before it can send data

  • Best suited when using a link medium such as a satellite link or undersea fiber optic link
  • Might take too long or too costly to detect a collision

Chapter 4–Internetworking Layer (IP)

STATION – where a decision is made as to which output link your packet will be forwarded on (ROUTERS – make sure the package arrives on the correct outbound link [ 5-20 routers])

CORE ROUTER – forwarding traffic within the core of the internet

EDGE ROUTER – provides a connection between local network & internet. Equivalent to GATEWAY

INTERNET PROTOCOL ADDRESS (IP) – ultimate destination address marked on each package

  • Old classic IPv4: four numbers (0-255) separated by dots
  • Ie. 212.78.1.25
  • IPv6 ie. 2001:0db8:85a3:0042:1000:8a2e:0370:7334

First part: two-piece address called NETWORK NUMBER

Second part: Host identifier address

TWO-CONNECTED NETWORK – two independent paths from a source network to a destination network in the core of the network

TRACEROUTE (TRACERT) – network diagnostic tool that allows you to trace the route between your computer and a destination computer

  • Takes advantage of a feature in the IP network protocol designed to avoid packets becoming “trapped” in the network (looks for potential traps and ways to solve the problem)

TIME TO LIVE (TTL) – starts out with a value of 30 but each time an IP packet is forward, -1 from TTL value

DYNAMIC HOST CONFIGURATION PROTOCOL (DHCP) – ability for your computer to get a different IP address when it is moved from one network to another

NON-ROUTABLE – will never be used as real addresses that will route data across the core network. Can be used on a single local network != on global network (start with 192.168)

NETWORK ADDRESS TRANSLATION (NAT)

  • Conserve the real routable address and use non-routable addresses over and over for workstations that move from one network to another

INTERNET SERVICE PROVIDER (ISP) – assigns you network numbers by giving you a portion of the network numbers they received from a higher-level internet service provider

PACKET VORTEX – an error situation where a packet gets into an infinite loop because of errors in routing tables

EXCEPTION: a signal that a condition has occurred that can’t be easily handled using the normal flow-of-control of a python program

  • Often defined as being “Errors”
  • However, no all exceptions are errors

Flow-of-control

  • For if-statements, only one of several statement blocks is executed and then flow-of-control jumps to the first statement after the if-statement.
  • For loops, when the end of the loop is reached, flow-of-control jumps back to the start of the loop and a test is used to determine if the loop needs to execute again. If the loop is finished, flow-of-control jumps to the first statement after the loop.
  • For function invocations, flow-of-control jumps to the first statement in the called function, the function is executed, and the flow-of-control jumps back to the next statement after the function call.

Principle 1: If a condition can be handled using the normal flow-of-control, don’t use an exemption

Principle 2: If you call a function that potentially raises exceptions, and you can do something appropriate to deal with the exception, then surround the code that contains the function call with a try: except: block

Principle 3: If you call a function that potentially raises exceptions, and you can’t do anything meaningful about the conditions raised, then don’t catch the exception message(s)

Catch all exceptions

try:

  # Your normal code goes here.

  # Your code should include function calls which might raise exceptions.

except:

  # If any exception was raised, then execute this code block.

Catch a Specific Exception

try:

  # Your normal code goes here.

  # Your code should include function calls which might raise exceptions.

except ExceptionName:

  # If ExceptionName was raised, then execute this block.

Catch Multiple Specific Exceptions

try:

  # Your normal code goes here.

  # Your code should include function calls which might raise exceptions.

except Exception_one:

  # If Exception_one was raised, then execute this block.

except Exception_two:

  # If Exception_two was raised, then execute this block.

else:

  # If there was no exception then execute this block.

Clean-up After Exceptions (finally code block)

try:

  # Your normal code goes here.

  # Your code might include function calls which might raise exceptions.

  # If an exception is raised, some of these statements might not be executed.

finally:

  # This block of code will always execute, even if there are exceptions raised

  • Selector: points to the HTML element you want to style
  • Declaration Block: contains one or more declarations separated by semicolons
  • CSS property (name, value, separated by colon)
  • CSS declaration always ends with a semicolon
  • Declaration block surrounded by curly braces

  • CSS Selectors: used to “find” (or select) HTML elements based on their element name, id, class, attribute, and more
  • Element selectors: selects elements based on the element name
  • id Selector: uses id attribute of HTML element ro select a specific element

#para1 {

  text-align: center;

  color: red;

}

  • Class selector: selects elements with a specific class attribute
  • .(name of class)

.center {

  text-align: center;

  color: red;

}

  • HTML elements can also refer to more than one class
  • Group selectors: separate selector with a comma

h1, h2, p {

  text-align: center;

  color: red;

}

  • CSS Comments: used to explain the code & help when you edit the source code at a later date
  • Ignored by browser
  • /*  …. */ (can span multiple lines)

[c]GLOSSARY

asymmetric key: An approach to encryption where one (public)

key is used to encrypt data prior to transmission and a different

(private) key is used to decrypt data once it is received.

certificate authority: An organization that digitally signs public

keys after verifying that the name listed in the public key is actually the person or organization in possession of the public key.

ciphertext: A scrambled version of a message that cannot be

read without knowing the decryption key and technique.

decrypt: The act of transforming a ciphertext message to a plain

text message using a secret or key.

encrypt: The act of transforming a plain text message to a ciphertext message using a secret or key.

plain text: A readable message that is about to be encrypted

before being sent.

private key: The portion of a key pair that is used to decrypt

transmissions.

public key: The portion of a key pair that is used to encrypt

transmissions.

shared secret: An approach to encryption that uses the same

key for encryption and decryption.

SSL: Secure Sockets Layer. An approach that allows an application to request that a Transport layer connection is to be encrypted as it crosses the network. Similar to Transport Layer Security (TLS).
